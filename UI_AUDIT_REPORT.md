# Отчет об аудите интерфейса и план обновлений

## Текущий статус
Я разработал современный лендинг с использованием заданной красно-белой цветовой схемы.
Однако, вы задали вопрос: "ты для всего сделал дизайн? для каждой странички?"
Проект представляет собой полноценное приложение с множеством страниц (дашборд, курсы, расписание, профиль, настройки и т.д.).

Мой текущий подход был сфокусирован на **лендинге** (странице входа/презентации). Остальные страницы приложения в данный момент используют прежний, сине-голубой дизайн (`dashboard/page.tsx` подтверждает это использованием специфичных CSS-классов вроде `bg-indigo-900` или `text-indigo-200`).

## План действий для приведения всего проекта к единому стилю

Чтобы сделать весь проект профессиональным и выдержанным в красно-белой гамме, необходимо:

1.  **Обновить `globals.css`**: Сделать переменные темы универсальными для всего приложения, а не только для лендинга.
2.  **Обновить компоненты оболочки (`Shell`, `Sidebar`)**: Они определяют общую структуру всех страниц приложения.
3.  **Рефакторинг страниц**: Обновить все страницы в `web/app/` (курсы, задания, настройки и т.д.), чтобы они использовали обновленные переменные темы и современные компоненты вместо старых стилей.
4.  **Унификация компонентов**: Привести `AttendanceChart`, `Metrics`, `AttendanceHeatmap` к новому стилю (красные акценты вместо синих).
